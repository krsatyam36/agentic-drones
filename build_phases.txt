PHASE 2 — SCHEMA + VALIDATOR (3-Layer Guardrail)
────────────────────────────────────────────────────────────────────────────
STATUS: ✅ DONE

Goal:
  Define a strict Pydantic schema for mission plans and build a 3-layer
  validation pipeline that catches bad LLM output before it reaches the drone.

Files:
  - src/schema/mission.py       Pydantic v2 models (MissionPlan, Action, Waypoint)
  - src/validator/validator.py  3-layer validation

3 Validation Layers:
  Layer 1 (Structural):   JSON parse, root object, actions field, non-empty list
  Layer 2 (Semantic):     altitude 1-50m, speed 0.5-15m/s, must-start-TAKEOFF,
                          must-end-LAND/RTL, geofence, jump distance ≤500m, max 50 actions
  Layer 3 (Re-prompt):    Errors fed back to LLM for self-correction (up to 3 retries)

Gate Test:
  python3 scripts/test_validator.py → 7/7 tests pass

Tests:
  - Valid perimeter loop accepted
  - Unsafe mission (500m altitude) rejected
  - Malformed JSON rejected
  - No TAKEOFF rejected
  - No LAND/RTL rejected
  - Geofence compliance checked
  - Speed out of bounds rejected

PHASE 3 — DETERMINISTIC EXECUTOR (Kill-the-LLM)
────────────────────────────────────────────────────────────────────────────
STATUS: ✅ DONE

Goal:
  Build a deterministic, auditable state machine that reads validated JSON
  and issues concrete flight commands. Must work WITHOUT the LLM running.

Files:
  - src/executor/executor.py

State Machine:
  IDLE → ARMING → TAKEOFF → EXECUTING(action_i) → ... → LANDING → DONE
                            |
                            ↓
                         FAILSAFE

Properties:
  - File-based: --mission validated.json, no LLM dependency
  - Every command logged with ISO 8601 timestamp
  - Deterministic: same JSON → identical audit log every time
  - 5s connection timeout, falls back to simulation mode if no drone

Gate Test:
  python3 scripts/test_executor_determinism.py → 2/2 tests pass
  python3 -m src.executor.executor --mission config/perimeter_loop.json

Results Verified:
  - Run 1 and Run 2 produce identical audit logs (minus timestamps)
  - All 10 actions (ARM → TAKEOFF → 8× GOTO → LAND) logged correctly
  - Final state: DONE

STATUS: ✅ DONE

  Support both local (Ollama) and cloud (OpenAI) LLM backends.

Files:

  - src/cli/cli.py         Interactive prompt loop

Architecture:
  - LLM proposes (generates JSON), never flies
  - Validator polices (3-layer guardrail)
  - Executor executes (deterministic, file-based)
  - Re-prompt loop: errors → LLM correction (up to 3 retries)

PHASE 5 — VISION AI (Challenge 3: Target Detection + Follow)
────────────────────────────────────────────────────────────────────────────
STATUS: ✅ DONE

Goal:
  Give the drone vision capability: detect a user-defined target via camera,
  save image and alert operator on first detection, then follow autonomously.

Files:
  - src/vision/vision_node.py       YOLO detection + PID servoing + mock fallback
  - src/vision/sim_camera.py        Synthetic camera (bouncing red circle target)

Implementation:
  - YOLOv8 object detection with configurable target class (person, car, dog, etc.)
  - PID visual servoing: pixel error → body velocity commands
  - On first detection: saves annotated frame, alerts operator
  - Target-lost fallback: hover (zero velocity) + yaw search (5 deg/s)
  - Mock detection fallback (HSV color tracking) when YOLO unavailable
  - Simulated camera: generates frames with a bouncing colored circle
    (no physical camera needed for demo)
  - Executor integration: --executor flag → calls set_velocity_body()
    (MAVSDK offboard velocity control in real mode, simulation mode fallback)

New CLI Arguments:
  --sim          Use simulated camera feed (synthetic moving target)
  --executor/-e  Pipe PID velocity commands to executor (offboard control)
  --frames N     Run for N frames then exit (for headless testing)
  --headless     No display window (runs in background)
  --connection   MAVSDK connection string (default: udp://:14540)

Gate Test (headless):
  python3 -m src.vision.vision_node person --sim --executor --frames 30 --headless

Documentation Updated:
  - README.md: updated Vision AI section with new flags, updated "Running the Challenges"
  - docs/ARCHITECTURE.md: added sim_camera, two detection modes, PID math, executor integration

Integration Results Verified:
  - Simulated camera: 640×480, red circle bouncing within frame boundaries
  - Mock detection: tracks the red circle via HSV thresholding
  - PID controller: pixel error (cx-320, cy-240) → body velocity (vx, vy, vz)
  - Executor integration: 30 SET_VELOCITY_BODY commands logged over 30 frames
  - First detection: image saved to output/detections/detection_*.jpg
  - Operator alert: "Target 'person' acquired. Image saved: ..."
  - Clean shutdown: LAND command issued, audit log saved
  - All 10/10 verify.sh tests still pass
  - Full command sequence: ARM → TAKEOFF → 30× SET_VELOCITY_BODY → LAND
